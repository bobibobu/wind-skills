# Conference Call Takeaway Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于从业绩会或类似管理层交流材料中提炼真正有价值的新信息，重点回答“管理层到底新增说了什么、哪些语气和措辞值得警惕、分析师最追问什么、哪些内容会改变后续研究重点”。输出重点不是会议逐段转述，而是帮助使用者抓住会后最值得更新的判断。

## 执行流程

### Step 1：确认会议范围与阅读目标

先确认对应的是哪次业绩会、投资者交流会或经营更新会，并明确用户最关心的用途：

- 快速复盘会议重点
- 判断管理层说法是否强化原观点
- 提炼团队分享纪要
- 寻找会后需要继续验证的风险点

若用户未说明，默认按“会后快速更新研究判断”展开。

### Step 2：先抓新增信息，再看重复表述

会议材料通常包含大量重复表述，应优先筛出真正有新增价值的部分：

- 对业绩、需求、订单、利润率等的边际更新
- 对前次未明确问题的补充回答
- 对市场争议点的正面回应或回避
- 对未来节奏、优先事项、资本配置的最新表态

如果某段内容只是重复公开口径，应降级处理，避免占据核心摘要。

### Step 3：识别管理层语气与信号变化

除了字面内容，还要关注表达方式变化，包括：

- 相比上次更乐观、更谨慎或更防守
- 对某类问题更愿意展开，还是刻意收缩
- 对风险描述更具体，还是用笼统表述遮掩不确定性
- 是否出现前后不一致、模糊化或转移焦点的情况

这一步的目标是捕捉“说了什么”和“怎么说”之间的差异。

### Step 4：提炼问答环节的真正焦点

分析师或投资者在问答里反复追问的内容，通常更接近市场核心分歧。应重点总结：

- 被反复追问的变量是什么
- 管理层回答是否直接、充分、可验证
- 哪些问题被回答得很强，哪些仍然含糊
- 是否有某个问题暴露出管理层准备不足或市场担忧加深

### Step 5：沉淀会后更新与警讯

在会议读完后，需要明确：

- 哪些原有观点被强化
- 哪些新信息要求修正研究框架
- 哪些风险虽未完全暴露，但已经出现苗头
- 后续最需要继续核验的经营变量是什么

### Step 6：整合成会后摘要成稿

最终输出应是一份可直接转发、可直接用于内部讨论的会后提炼，不展示逐段阅读过程或原始摘抄堆叠。

---

## 质量要求

1. 必须优先呈现新增信息，不把会议重复内容当作重点。
2. 不能只摘抄管理层原话，要转化成研究意义。
3. 问答环节的焦点和回答质量必须单独处理，因为它通常最接近市场分歧。
4. 语气变化要结合上下文判断，避免把正常措辞差异误判为趋势变化。
5. 输出应服务“会后更新观点”，而不是沦为冗长会议纪要。

# Major Announcement Impact Skill

## 数据依赖与执行前置条件

本技能依赖 Wind 数据源。执行本技能前，必须先检测 `wind-mcp-skill` 是否已安装且可用。

可用性判断至少包括：

- `wind-mcp-skill` 已安装；
- 其 CLI / MCP 调用入口可执行；
- `WIND_API_KEY` 已配置；
- 能完成一次最小 Wind 查询并返回有效数据。

若 `wind-mcp-skill` 已安装且可用，先通过该技能获取并整理行情、K 线、成交量、换手率、指数环境、板块表现、公告新闻等必要数据，再进入分析或筛选。

若未安装 `wind-mcp-skill`，先向用户推荐安装，并说明缺少 Wind 数据源会影响分析结论的可靠性。

推荐安装命令：

```bash
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y
```

若 GitHub 源不可用，使用 Gitee 镜像：

```bash
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```

若已安装但不可用，应说明具体原因，例如缺少 `WIND_API_KEY`、CLI 不可执行、接口返回错误或额度不足，并先协助用户补齐数据条件。

若尚未取得足够的 Wind 数据，不应直接给出确定性结论；只能输出框架性判断、所需数据清单，或等待数据补齐后再分析。

## 技能定位

本技能用于分析重大公告事件对公司、股东和市场预期的实际影响，重点回答“这条公告改变了什么、影响会先落在哪一层、谁受益谁受损、后续还有哪些关键不确定性”。输出重点不是把公告摘要改写一遍，而是把影响链条拆解清楚。

## 执行流程

### Step 1：确认公告类型与核心事实

先确认公告属于哪类重大事项，例如：

- 并购、出售、分拆、重组
- 减持、增持、定增、配股、可转债
- 重大合同、订单、合作、诉讼、监管处罚
- 产能建设、资产处置、控制权变化

同时梳理关键事实：谁参与、金额多大、时间安排、条件限制、是否仍需审批。

### Step 2：建立事件前的基准状态

事件影响不能脱离原有背景判断。需要先明确：

- 公司在公告前处于什么经营和估值状态
- 市场原本最担心或最期待的是什么
- 这件事是否早有传闻或预期
- 该事件是补强主线，还是改变主线

### Step 3：拆解直接影响与二阶影响

围绕公告本身，分层判断：

- 对收入、利润、现金流、资产负债表的直接影响
- 对控制权、治理、融资能力、股东回报的结构性影响
- 对行业地位、竞争格局、战略方向的中期影响
- 对市场情绪和估值锚点的短期影响

不要把所有后果混在一起，应区分立即可见与后续传导。

### Step 4：识别谁受益、谁受损、谁仍在观望

重大公告往往不是单边影响，需要明确：

- 公司本身是否直接受益
- 原股东、新投资方、管理层、债权人或供应链谁受到影响
- 对同行、上下游或题材板块是否存在外溢效应
- 哪类利益相关方的态度最可能影响事件落地

### Step 5：评估不确定性与落地门槛

进一步说明这类事件为什么还不能轻易下定论，重点包括：

- 审批、交割、执行节奏风险
- 价格、估值、对价结构是否存在争议
- 市场是否可能高估短期收益、低估长期成本
- 哪些条件若未满足，原有解读需要推翻

### Step 6：整合成事件影响成稿

最终输出应帮助使用者快速完成“事件定性 + 影响拆解 + 风险提示”，不展示内部检索路径。

## 质量要求

1. 重大公告必须先讲事实，再讲影响，避免先入为主。
2. 直接影响与二阶影响要分层处理，不能混成一句泛论。
3. 不能把“讲故事的空间”误写成“已经兑现的价值”。
4. 若事件仍需审批或交割，应把落地风险放在核心位置。
5. 输出要帮助使用者判断这是一条可以交易、跟踪还是暂时回避的事件。

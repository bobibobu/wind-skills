# Bull Bear Case Builder Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于围绕一只股票的核心命题，系统构建看多与看空两套论证，并比较双方证据、假设、催化与脆弱点。输出重点不是“站队”，而是帮助使用者识别当前最值得争论的变量，压缩情绪化判断和确认偏误。

## 执行流程

### Step 1：定义争论命题与讨论边界

先明确这次论证到底在回答什么问题，例如：

- 这只股票未来 6-12 个月是否有超额收益机会
- 当前股价是否高估或低估了某项业务变化
- 近期上涨是趋势启动还是情绪透支

同时界定时间维度、讨论对象和可接受的证据范围，避免把长期逻辑与短期交易混在一起。

### Step 2：收集双方共用的基础事实

先整理不带立场的基础信息，作为后续多空两边共同使用的底座，包括：

- 公司业务与产业链位置
- 最近的经营变化、公告、财务表现
- 市场关注的核心变量
- 当前估值和股价位置

这一阶段不急于下结论，重点是把双方都会引用的事实材料整理齐。

### Step 3：搭建看多论证

围绕最强的上涨逻辑，建立一条完整链路：

- 市场可能低估了什么
- 哪个变量正在改善
- 该改善为何有持续性
- 一旦被验证，股价弹性从哪里来

看多论证应尽量具体到经营变量、事件节点和预期修正路径，而不是只写“行业空间大”“公司优秀”这类空泛表达。

### Step 4：搭建看空论证

围绕最强的反方逻辑，建立对应链路：

- 当前市场可能高估了什么
- 哪些改善可能只是阶段性噪音
- 哪些约束会阻碍逻辑兑现
- 若不及预期，估值和情绪会如何反噬

看空论证不能沦为机械挑刺，而要聚焦真正会导致投资失败的关键变量。

### Step 5：比较证据强弱与触发条件

把多空双方放到同一框架下比较，重点看：

- 哪边的证据更直接、更可验证
- 哪边更依赖未经证实的外推假设
- 哪些事件会率先验证某一方
- 哪些变量一旦反向，就会让整套叙事失效

必要时可区分“短期谁占优”和“长期谁占优”，避免强行给出单一时间维度的答案。

### Step 6：形成结论与后续跟踪清单

最终给出：

- 当前更占优的是哪一边，理由是什么
- 这个结论的置信度有多高
- 接下来最需要跟踪的验证点是什么

若双方证据仍接近，也应明确写出“暂不站队，等待验证”的条件，而不是假装已有清晰答案。

## 质量要求

1. 多空双方必须使用同一组事实底座，避免“双重标准”。
2. 看多和看空都要写最强版本，而不是把反方写成稻草人。
3. 结论必须落到具体变量与验证条件，不能只给模糊态度。
4. 若某一方证据明显薄弱，应直接指出，不做表面平衡。
5. 输出目标是帮助判断分歧，不是写成情绪化的站队文章。

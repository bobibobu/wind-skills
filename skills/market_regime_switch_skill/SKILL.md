# Market Regime Switch Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于判断市场当前所处的整体状态，回答“现在是适合进攻、适合防守、适合精选个股，还是需要降低节奏；市场状态是否正在切换；这种切换由什么驱动”。输出重点不是一句简单的看多或看空，而是给出市场分档、证据框架、切换线索和仓位含义。

## 执行流程

### Step 1：确认观察窗口与决策目的

先明确本次判断服务什么决策：

- 是为了判断总仓位
- 是为了选择成长、价值、周期或防御风格
- 是为了周度策略更新
- 是为了应对近期状态变化带来的节奏问题

同时确认观察窗口是短线、中线还是阶段性框架。若用户未说明，默认按“当前阶段市场状态”展开。

### Step 2：建立市场状态判断框架

市场分档不能只看指数涨跌，需要综合以下层面：

- 指数趋势与关键位置
- 市场广度和参与度
- 领涨方向是否清晰
- 情绪温度与拥挤程度
- 宏观环境、流动性与政策背景
- 盈利预期或财报主线是否支持当前行情

这一阶段的目标是构建一个足以支持分档判断的完整框架。

### Step 3：进行市场状态分类

在证据充分后，将市场大致分为以下状态之一：

- 进攻阶段：风险偏好上升，主线清晰，扩散良好
- 防守阶段：避险偏好抬升，资金集中在低波动方向
- 震荡阶段：主线不清，轮动快，分化大
- 切换阶段：原有主线松动，新方向尚在形成

分类时必须给出依据，而不是只下结论。

### Step 4：解释状态形成的主因与次因

进一步说明为何会形成当前状态，优先拆解：

- 宏观与政策环境是否支持风险偏好
- 资金面和风格偏好是否发生变化
- 广度、情绪与主线是否相互验证
- 若状态正在切换，切换的起点来自哪里

### Step 5：识别状态切换信号与失效条件

阶段判断最重要的是可跟踪，因此需要明确：

- 哪些信号能确认当前状态继续成立
- 哪些变化意味着状态可能切换
- 哪些风险会使当前判断快速失效
- 若判断失效，市场更可能切到哪一档

### Step 6：形成市场状态判档成稿

最终输出应帮助使用者迅速知道“现在该用什么姿态面对市场”。正文只保留状态分类、证据说明、切换信号与仓位层面启发。

## 质量要求

1. 市场状态判断必须基于多维证据，不能只由指数涨跌决定。
2. 要明确当前档位、形成原因和切换条件三个层次。
3. 需把广度、情绪、主线和宏观背景放在同一框架下，提升一致性。
4. 若状态尚不清晰，应明确写出“切换中”或“震荡中”，不能强行归类。
5. 输出要能直接服务总仓位和风格选择，而不是停留在宏观概括。
